# 153.1 — Remove generative tokens toggle from campaign review

## Description

The campaign review page currently shows **"Use generative tokens?"**, which lets players flip the flag after world generation. Product rule: that preference is chosen only at **campaign start**. Remove the review checkbox (and its saver wiring); keep the start-form control and persistence at create time.

## Acceptance criteria

- [ ] Campaign review UI does not render the "Use generative tokens?" checkbox
- [ ] Campaign start form still exposes "Use generative tokens?" (default OFF)
- [ ] Review no longer calls `editGenerativeTokens` from the page savers
- [ ] Unit test covers that review does not mount the toggle
- [ ] `npm test` / `npm run lint` / `npm run build` / `npm run deadcode` pass
- [ ] `act` pr-checks.yml + deadcode.yml succeed
