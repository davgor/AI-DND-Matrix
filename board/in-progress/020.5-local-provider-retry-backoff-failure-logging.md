# Local-provider retry/backoff + failure logging

## Description
Ensure transient local-provider failures are retried with backoff and terminal failures surface typed, sanitized errors in logs — with parity to how other providers behave under `withRetry`.

**Current state:** `campaignIpc` wraps the selected provider with `withRetry` (3 attempts, backoff) for all modes including `llamacpp`. What remains is llama-specific error typing and richer sanitized diagnostics (lifecycle state, attach vs managed).

## Acceptance Criteria
- [x] Failing provider calls are retried a small fixed number of times with backoff before giving up (global `withRetry`)
- [ ] Non-retryable errors (e.g. invalid config / `LlamaCppLifecycleError` category `config`) fail fast without retry delay
- [ ] After retries are exhausted, a typed local-provider-unreachable error is surfaced (distinct from generic `ProviderUnreachableError` if useful for UI), not an uncaught exception
- [ ] Failure events are written to the local log file with sanitized diagnostics (provider name, host/port, lifecycle state, retry count, error class) and no secrets
- [ ] Unit tests cover transient failure then success, repeated failure then typed terminal error, and immediate fail-fast on non-retryable config errors
