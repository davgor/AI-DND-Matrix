# Validation/error UX + redaction rules

## Description
Implement consistent validation and error messaging for provider settings with strict secret redaction.

## Acceptance Criteria
- [ ] Settings validation errors are field-specific and actionable
- [ ] Runtime/connection test errors are mapped to user-friendly messages
- [ ] Logs and UI messages never include raw API keys or sensitive filesystem tokens
- [ ] Tests verify redaction behavior across success/failure paths
