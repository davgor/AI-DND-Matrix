# m004.5 Post-campaign map generation (LLM plan → asset compose)

## Description
After the standard sprite-campaign stages (world, canon, regions, NPCs, story), add map-generation stages: LLM proposes map plans and placements from generated world content; a deterministic composer builds grids from pregenerated tiles/buildings/furniture and validates solvability.

## Acceptance Criteria
- [ ] Sprite create pipeline runs shared stages first, then map-plan + compose + persist stages (documented in create checklist)
- [ ] LLM output is a typed plan (rooms/zones, theme tags, prop intents, spawn intents) — not raw tile matrices as source of truth
- [ ] Composer materializes walkable grids with real walls and reachable spawns/transitions; invalid plans fail with typed errors
- [ ] Generated maps are grounded in existing regions/NPC roster (no orphan world that ignores campaign content)
- [ ] Contract/fixture tests cover plan normalize + compose for a realistic multi-map fixture
