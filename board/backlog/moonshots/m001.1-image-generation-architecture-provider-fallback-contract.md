# m001.1 Image generation architecture + provider fallback contract

## Description
Define the architecture and provider contract for image generation with deterministic local/cloud fallback behavior.

## Acceptance Criteria
- [ ] A provider-agnostic image generation interface is defined with typed request/response schema
- [ ] Local-first and cloud fallback rules are explicit and deterministic
- [ ] Fallback does not duplicate requests or create conflicting assets
- [ ] Typed error categories exist for provider unavailable, timeout, validation, and policy rejection
