# m002.1 Multiplayer architecture and authority model

## Description
Define the host-authoritative multiplayer architecture and responsibilities across host and guest clients.

## Acceptance Criteria
- [ ] Authority model is documented: host authoritative, guests non-authoritative
- [ ] Event ownership rules are explicit for chat, actions, and turn progression
- [ ] Guest identity to party-member identity binding ownership is defined and host-authoritative
- [ ] Failure domains are identified (host crash, guest disconnect, network partition)
- [ ] Typed protocol contract exists for host/guest message categories
