# m005.5 Thin remote client play surface

## Description
Define the minimum remote UI needed to play: Scene/Social consumption, action input, and essential sheet/combat affordances — without cloning the full Electron chrome.

## Acceptance Criteria
- [ ] MVP screen list is documented (play stream, action box, combat essentials, connection status)
- [ ] Mobile-first layout constraints are noted (single column; no dual-pane requirement)
- [ ] Client is explicitly thin: no local rules engine authority, no local provider keys required for Player2/local path
- [ ] Packaging options are listed (PWA, Capacitor shell, or in-app webview) with a recommended MVP choice
- [ ] Offline-on-phone behavior is defined as “cannot play until PC reachable,” not a second save universe
