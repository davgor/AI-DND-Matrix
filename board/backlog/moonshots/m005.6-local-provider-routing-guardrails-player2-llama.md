# m005.6 Local-provider routing guardrails (Player2 / llama)

## Description
Ensure remote play only uses the PC’s configured local providers (Player2, llama.cpp when available, or other local OpenAI-compatible endpoints) and never turns the PC into an open LLM proxy.

## Acceptance Criteria
- [ ] Remote session inherits PC provider settings; remote client cannot override to a cloud key on the host without an explicit separate product decision
- [ ] All agent completions for a remote-play session are issued from the PC process via existing provider adapters
- [ ] Direct remote access to `localhost:4315` (or llama bind address) is forbidden; only mediated game/tool APIs are exposed
- [ ] Provider-down UX is defined (clear remote error; no silent fallback to developer-paid cloud)
- [ ] Compatibility matrix documents Player2 now and llama.cpp as a dependent follow-on (epic 020)
