# m002.12 Moonshot validation runbook and go/no-go metrics

## Description
Define validation scenarios and decision metrics for promoting host-driven multiplayer into the main backlog.

## Acceptance Criteria
- [ ] Runbook includes host online/offline transitions, guest join via command, chat/action load, and reconnect scenarios
- [ ] Metrics include join success rate, interaction latency, host AI routing reliability, and status-indicator accuracy
- [ ] Metrics include guest-to-party-member mapping consistency across reconnect/restart and AI fallback personality continuity quality checks
- [ ] Metrics include `ai-able` enforcement correctness (no unauthorized AI control when disabled) and goofy-absence fallback correctness when guest is absent
- [ ] Go/no-go thresholds are explicit and tied to user experience risk
- [ ] Validation report template exists for recommendation and next-step scope
