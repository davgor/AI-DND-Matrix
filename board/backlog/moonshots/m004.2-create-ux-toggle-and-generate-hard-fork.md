# m004.2 Create UX toggle and generate hard-fork

## Description
Add a create-flow toggle for narrative vs sprite world type. Choosing Generate hard-forks campaign creation and subsequent review/hub/play entry so sprite and narrative do not share an incompatible play shell.

## Acceptance Criteria
- [ ] Campaign start UI exposes a clear narrative/sprite world-type toggle before Generate
- [ ] Selected `worldType` is sent on create and stored on the campaign
- [ ] After Generate, sprite campaigns enter sprite-capable review/hub/play paths; narrative paths remain unchanged
- [ ] Switching the toggle before Generate does not leave mixed half-state from a prior attempt
- [ ] Automated or component test covers toggle → create payload `worldType`
