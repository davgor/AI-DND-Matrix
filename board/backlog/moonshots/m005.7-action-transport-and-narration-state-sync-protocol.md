# m005.7 Action transport and narration/state sync protocol

## Description
Specify the wire protocol for player actions, streaming Scene/Social narration, combat prompts, and sheet/state projections between PC host and remote client.

## Acceptance Criteria
- [ ] Message types cover: auth, action submit, narration stream chunks, turn resolved snapshot, error, heartbeat
- [ ] Streaming semantics match existing play UX enough to feel continuous (not poll-only walls of text)
- [ ] State sync strategy is defined (full snapshot vs deltas) with versioning to avoid stale UI
- [ ] Latency budget targets are stated for action→first token and action→turn resolved on LAN vs tunnel
- [ ] Protocol is transport-agnostic enough to ride LAN WebSocket, tunnel HTTPS, or relay multiplex
