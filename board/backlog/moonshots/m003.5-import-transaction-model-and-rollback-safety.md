# M003.5 - Import transaction model and rollback safety

## Description
Ensure mod imports are atomic and recoverable, even on partial failure.

## Acceptance Criteria
- [ ] Import runs in transaction-safe stages with clear commit boundaries
- [ ] Failed imports do not leave partial broken records
- [ ] Rollback/retry flow is documented and test-covered
- [ ] Import summary includes inserted/updated/skipped/error counts
