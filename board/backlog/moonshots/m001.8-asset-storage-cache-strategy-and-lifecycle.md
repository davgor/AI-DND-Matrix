# m001.8 Asset storage/cache strategy and lifecycle

## Description
Define storage and cache behavior for generated world/character image assets to balance persistence and disk usage.

## Acceptance Criteria
- [ ] Asset location strategy is defined (local storage path, naming, linkage keys)
- [ ] Cache strategy supports fast reuse of recent valid images
- [ ] Asset invalidation/cleanup policy exists and avoids deleting in-use assets
- [ ] Startup/reload can resolve missing assets with safe fallback behavior
