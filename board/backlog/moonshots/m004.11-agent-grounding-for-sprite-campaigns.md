# m004.11 Agent grounding for sprite campaigns

## Description
Adapt DM/NPC agent context for sprite campaigns: compact map/room summaries, nearby entities, transition facts, and explicit bans on inventing walls, tiles, or positions. Lazy-bind and dialogue still re-ground from SQLite each call.

## Acceptance Criteria
- [ ] Sprite turn/talk prompts include engine-derived map summary (current map, coords, visible/adjacent entities)
- [ ] Agents never supply authoritative coordinates or rewrite collision; invalid proposals are ignored/clamped by engine
- [ ] Dialogue with newly bound NPCs uses persisted identity after lazy bind
- [ ] Narrative campaign agent prompts remain unchanged when `worldType` is narrative
- [ ] Unit or contract tests assert sprite context fields are present and position fields are engine-sourced
