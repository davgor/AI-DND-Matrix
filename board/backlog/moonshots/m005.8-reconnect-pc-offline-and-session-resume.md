# m005.8 Reconnect, PC offline, and session resume

## Description
Define behavior when the remote client drops, the PC sleeps/quits, or the tunnel/relay flaps — without corrupting SQLite saves or double-applying turns.

## Acceptance Criteria
- [ ] In-flight turn handling is idempotent (reconnect does not duplicate engine resolution)
- [ ] Remote UX clearly distinguishes “reconnecting,” “PC offline,” and “session revoked”
- [ ] Resume after brief disconnect restores stream/state without requiring full re-pair when device token is valid
- [ ] PC sleep/quit forces remote into a safe idle state; saves remain consistent with last committed turn
- [ ] Documented test scenarios cover mid-narration disconnect and mid-combat disconnect
