# m002.10 Data persistence model for multiplayer session state

## Description
Define persistence boundaries for multiplayer campaign/session data so host authority and replay consistency are maintained.

## Acceptance Criteria
- [ ] Multiplayer session metadata model is defined (host id, session id, participants, status)
- [ ] Persistence model stores stable guest-to-party-member identity mappings
- [ ] Persistence model stores per-mapped-party-member `ai-able` preference with effective timestamps
- [ ] Event/action logs persist with ordering identifiers suitable for replay/debugging
- [ ] Interaction history features needed to inform AI fallback personality/behavior are persisted with clear retention rules
- [ ] Non-`ai-able` absence events persist goofy DM reason text and session linkage for continuity/recap
- [ ] Persistence writes are host-authoritative and crash-safe
- [ ] Restart recovery can restore last known stable session state
