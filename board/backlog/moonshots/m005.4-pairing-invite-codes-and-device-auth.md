# m005.4 Pairing invite codes and device auth

## Description
Define low-friction pairing so a remote client can join a specific PC session without passwords, while remaining unusable by strangers who find a URL.

## Acceptance Criteria
- [ ] Short-lived invite code and/or QR pairing flow is specified
- [ ] Device token issuance after successful pair is defined (persist across reconnects until revoked)
- [ ] Invite expiry, single-use vs multi-use policy, and max concurrent remotes are explicit
- [ ] Revoke-all and revoke-one-device behaviors are defined in the PC UI contract
- [ ] Unauthenticated clients cannot invoke game or provider tools
