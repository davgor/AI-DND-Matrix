# m001.10 Failure handling, fallback, and deterministic recovery

## Description
Provide deterministic failure and fallback behavior for image generation across local/cloud provider paths.

## Acceptance Criteria
- [ ] Provider failure categories map to explicit retry/fallback actions
- [ ] Local-to-cloud and cloud-to-local fallback paths are deterministic and logged
- [ ] Recovery never corrupts campaign linkage or character consistency records
- [ ] Error UX remains actionable without exposing secrets or provider internals
