# M003.6 - Conflict resolution and version upgrade policy

## Description
Define deterministic behavior for content key collisions and mod version updates.

## Acceptance Criteria
- [ ] Conflict policy supports replace, keep-both, and ignore modes where appropriate
- [ ] Version upgrades can diff old vs new entries and apply deterministic outcomes
- [ ] Removal/deprecation behavior is defined for missing entries in new versions
- [ ] Operator-facing report explains what changed and why
