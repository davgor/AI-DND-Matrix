# m005.3 Reachability modes (LAN, BYO tunnel, optional relay)

## Description
Define how a remote client discovers and reaches the PC without requiring developer-hosted game servers: same-LAN, user-owned tunnel, and optionally a dumb shared relay for NAT traversal/pairing.

## Acceptance Criteria
- [ ] LAN mode is specified (discovery + connect on local network) with no cloud dependency
- [ ] BYO tunnel mode is specified (user configures Tailscale / Cloudflare Tunnel / equivalent; PC advertises reachable URL)
- [ ] Optional shared relay mode is specified as outbound-only from PC (no inbound port-forward required) and documented as non-authoritative byte-forwarding only
- [ ] Mode selection UX and failure messaging are sketched for each path
- [ ] MVP can ship with LAN + BYO tunnel alone (relay not mandatory for vetting go)
