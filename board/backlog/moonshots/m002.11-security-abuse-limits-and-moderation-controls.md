# m002.11 Security, abuse limits, and moderation controls

## Description
Add baseline controls for multiplayer safety, abuse prevention, and operational protection.

## Acceptance Criteria
- [ ] Rate limits/quotas exist for chat/action spam from guests
- [ ] Host has controls to remove/block disruptive participants
- [ ] Sensitive data (keys, tokens, internal endpoints) is never exposed to guests
- [ ] Security event logging exists for join failures and abuse actions
