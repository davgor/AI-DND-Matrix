# m004.1 Architecture and worldType contract

## Description
Define the sprite vs narrative campaign architecture: typed `worldType`, pipeline fork points, engine ownership of spatial truth, and renderer/agent boundaries so LLM planning never invents walkable geometry.

## Acceptance Criteria
- [ ] Shared contract documents `worldType: 'narrative' | 'sprite'` on campaign create and persisted campaign records
- [ ] Architecture note lists which stages are shared vs sprite-only (maps, spatial runtime, sprite play shell, FF combat UI)
- [ ] Engine owns tiles, collision, positions, transitions, and combat resolution; agents propose layout intent and narration only
- [ ] Failure domains are explicit (invalid map plan, missing asset ids, lazy-bind failure, combat desync)
- [ ] Typed IPC/shared types sketch exists for map state, entity refs, and sprite component assemblies
