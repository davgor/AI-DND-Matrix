# m001.7 Character consistency data model + persistence

## Description
Persist character visual identity data so appearances remain stable across scenes and sessions.

## Acceptance Criteria
- [ ] Data model stores portrait image reference, prompt seed/style metadata, and outfit/visual trait descriptors
- [ ] Stored consistency data is rehydrated on load and reused by generation pipeline
- [ ] Character updates preserve identity continuity unless explicitly changed
- [ ] Persistence logic is covered by tests for create/load/update/reload paths
