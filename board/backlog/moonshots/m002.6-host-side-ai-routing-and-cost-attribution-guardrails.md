# m002.6 Host-side AI routing and cost attribution guardrails

## Description
Ensure all AI provider usage in multiplayer sessions is routed through host authority and billed/attributed to host context.

## Acceptance Criteria
- [ ] Guest requests requiring AI are proxied through host-controlled provider path
- [ ] Guests cannot invoke paid provider endpoints directly through client settings
- [ ] Host-side logs/metrics can attribute AI usage to multiplayer campaign/session context
- [ ] Failure in host AI routing returns safe guest-facing errors without leaking secrets
- [ ] When a mapped guest is absent and `ai-able` is true, host-side AI fallback can reference that guest's prior interaction history to inform party-member behavior
- [ ] When `ai-able` is false, host-side AI must not control that party member and the system routes through DM goofy-absence handling instead
