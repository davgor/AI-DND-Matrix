# m005.2 PC remote-play agent and session lifecycle

## Description
Specify the PC-side remote-play agent (mode inside Electron or companion process): enable/disable remote play, bind to an active campaign/character session, and tear down cleanly.

## Acceptance Criteria
- [ ] Session lifecycle states are defined (off, advertising, paired, playing, suspended, stopped)
- [ ] Enabling remote play requires an explicit user action in the PC app
- [ ] Only one active remote-play session per campaign (or documented multi-session policy) is allowed
- [ ] Stopping remote play or quitting the app revokes paired clients and closes transports
- [ ] Agent reuse of existing main-process turn/provider pipeline is preferred over a parallel game loop
