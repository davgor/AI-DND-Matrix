# m005.9 Security, abuse limits, and endpoint isolation

## Description
Lock down remote play so a paired (or attacked) endpoint cannot exfiltrate saves wholesale, burn local GPU indefinitely, or pivot into the user’s LAN services.

## Acceptance Criteria
- [ ] Threat model covers: invite code guessing, stolen device token, malicious remote client, relay operator curiosity
- [ ] Rate limits exist for actions and provider-bound requests per session
- [ ] Scope isolation: remote API cannot reach arbitrary localhost ports beyond the game agent
- [ ] Sensitive settings (API keys, .env) are never sent to the remote client
- [ ] Logging/redaction policy avoids leaking narration secrets to shared relay operators (prefer end-to-end encrypted payloads when relay is used)
