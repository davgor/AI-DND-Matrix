# m005.12 Moonshot validation runbook and go/no-go metrics

## Description
Define validation scenarios and decision metrics for promoting PC-routed remote play into the main backlog.

## Acceptance Criteria
- [ ] Runbook covers: LAN pair + play turn, BYO tunnel pair + play turn, provider-down, PC quit mid-turn, reconnect resume, invite expiry, revoke device
- [ ] Metrics include pair success rate, action→first-token latency, turn-resolved success rate, reconnect success rate, and unauthorized-access attempt rejection rate
- [ ] Metrics include “no developer LLM spend” confirmation for the Player2/local path
- [ ] Go/no-go thresholds are explicit (playability, security, and hosting-cost constraints)
- [ ] Validation report template exists for recommendation and next-step scope (including m002 interaction)
