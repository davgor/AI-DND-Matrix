# M003.3 - Validation, normalization, and canonical keying

## Description
Validate imported entries and normalize into canonical, deduplicated DB-ready records.

## Acceptance Criteria
- [ ] Validation enforces required fields by type (spell/monster/NPC/gear)
- [ ] Normalization handles case, whitespace, tag aliases, and enum mapping
- [ ] Canonical key strategy prevents duplicate near-identical entries
- [ ] Validation report separates blocking errors from non-blocking warnings
