# m002.9 Reconnect, resume, and offline fallback behavior

## Description
Handle disconnect/reconnect flows for guests and host transitions to preserve clarity and session continuity.

## Acceptance Criteria
- [ ] Guest reconnect flow can resume active session context when host is still online
- [ ] If host goes offline mid-session, guests receive clear offline transition state
- [ ] Rejoin after host restart is possible with deterministic session compatibility checks
- [ ] Offline fallback does not create divergent non-authoritative campaign branches
- [ ] While a guest is absent but host remains online and `ai-able` is true, host can continue play with AI controlling that guest's mapped party member using stored interaction-informed behavior context
- [ ] While a guest is absent and `ai-able` is false, flow records and displays a goofy DM absence reason (for example bizarre abduction/portal event) instead of AI control
