# m004.6 NPC and ambient entity placement

## Description
Place campaign-generated NPCs onto maps, and allow additional ambient sprite entities without full identity yet. Placement must respect collision, region/map affinity, and spawn points from the composed maps.

## Acceptance Criteria
- [ ] Each campaign NPC can be assigned a map position (or marked unplaced with a deterministic placement pass)
- [ ] Ambient entities can exist as sprite assemblies (component ids) without a full NPC identity row yet
- [ ] Placement never puts entities inside walls or on transition tiles unless explicitly allowed
- [ ] Persistence reloads placed entities with stable ids across sessions
- [ ] Unit tests cover placement validation and ambient vs bound-NPC distinction
