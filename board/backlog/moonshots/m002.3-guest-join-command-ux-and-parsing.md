# m002.3 Guest join command UX and parsing

## Description
Add a simple command-based join flow so guests can join multiplayer campaigns quickly.

## Acceptance Criteria
- [ ] Guest can join via simple command pattern (for example: `/join <code>`)
- [ ] Command parser validates malformed/expired codes with actionable errors
- [ ] Join flow can be triggered from primary app UI with minimal clicks/typing
- [ ] Join success path lands guest into active campaign session context
- [ ] Join flow resolves guest to a specific party-member identity and preserves that mapping for future rejoins
- [ ] Join or post-join settings expose `ai-able` toggle for that mapped party member with clear behavior explanation
