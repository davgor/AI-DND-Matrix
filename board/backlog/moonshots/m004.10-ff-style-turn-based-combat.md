# m004.10 FF-style turn-based combat informed by existing rules

## Description
Replace free-text encounter UX for sprite campaigns with an old-school Final Fantasy–style turn-based combat loop. Resolution must use the existing rules engine (initiative, abilities, HP/AC, damage types, conditions, flee/yield where applicable), adapted to a grid/party battle presentation.

## Acceptance Criteria
- [ ] Provoking/engaging on the map enters a turn-based combat UI (menu/actions per turn), not narrative-only combat text as primary control
- [ ] Damage, initiative, checks, and death-mode hooks call existing engine APIs (no parallel unofficial math)
- [ ] Combatants reference party + map NPCs with persisted HP/conditions after combat ends
- [ ] Flee/defeat/victory outcomes return the player to the correct map position with consistent world state
- [ ] Engine/orchestration tests cover at least one full sprite-combat round trip (start → action → end)
