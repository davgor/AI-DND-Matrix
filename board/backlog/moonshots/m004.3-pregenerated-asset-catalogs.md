# m004.3 Pregenerated asset catalogs (tiles, props, sprite components)

## Description
Ship versioned, pregenerated pixel asset catalogs the LLM may only reference by id: terrain/wall tiles, buildings, furniture/props, and a sprite component book (body, hair, outfit layers, etc.) for composing characters.

## Acceptance Criteria
- [ ] Catalog schema defines stable string ids, layer/slot rules, and collision/footprint metadata for tiles and props
- [ ] Sprite component book defines slots, z-order, and compatibility rules; assemblies are lists of component ids
- [ ] Composer rejects unknown ids and incompatible slot stacks (deterministic validation)
- [ ] At least one minimal in-repo pack exists so generation and renderer can run offline without external downloads
- [ ] Unit tests cover id validation, slot conflicts, and collision metadata lookup
