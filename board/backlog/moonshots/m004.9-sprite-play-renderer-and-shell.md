# m004.9 Sprite play renderer and shell

## Description
Build the sprite campaign play shell: grid map view with camera, layered tiles/props, composed sprite entities, and input for move/interact — forked from the narrative four-column text play UI.

## Acceptance Criteria
- [ ] Sprite campaigns open a map-centric play view; narrative play UI is unused for `worldType: sprite`
- [ ] Renderer draws composed tiles and sprite component stacks from catalog ids
- [ ] Camera follows the player across multi-map loads without leaking the previous map's entities
- [ ] Move/interact input calls typed IPC and reflects engine-authoritative positions
- [ ] Basic render/smoke path documented for manual verification
