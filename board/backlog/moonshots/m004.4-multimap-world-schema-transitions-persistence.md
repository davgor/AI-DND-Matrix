# m004.4 Multi-map world schema, transitions, and persistence

## Description
Model a Pokemon-like multi-map world: named maps linked by transitions (doors, exits, overworld edges), each with a grid of composed tiles, entity spawn points, and durable save state for player position and visited maps.

## Acceptance Criteria
- [ ] DB/shared types persist maps, grid layers (or equivalent composed grid), transition links, and player `(mapId, x, y)`
- [ ] Transitions are bidirectional or explicitly one-way with validated landing tiles
- [ ] World graph can represent multiple interiors plus overworld-scale maps without requiring a single mega-grid
- [ ] Reload after restart restores map graph and player position identically
- [ ] Repository/engine tests cover transition travel and invalid landing rejection
