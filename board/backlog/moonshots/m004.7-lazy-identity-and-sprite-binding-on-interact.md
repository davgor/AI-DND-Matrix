# m004.7 Lazy identity and sprite binding on interact

## Description
Support deferred binding: entities may lack full identity and/or complete sprite component assignment until the player interacts. On first interact, generate/assign identity from sprite appearance + location context (and/or assign component outfit from NPC traits), then persist so later visits are stable.

## Acceptance Criteria
- [ ] Interacting with an unbound ambient sprite creates/binds a durable NPC identity grounded in component ids + map/location context
- [ ] Interacting with an identity-only NPC can assign and persist a sprite component assembly when missing
- [ ] Second interact never regenerates a different identity or outfit for the same entity id
- [ ] Failure to generate bind falls back deterministically without corrupting map state
- [ ] Tests cover first-bind persistence and idempotent re-interact
