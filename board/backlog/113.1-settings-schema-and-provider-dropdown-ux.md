# 113.1 Settings schema + provider dropdown UX

## Description
Replace the provider radio group with a dropdown and extend `ProviderSettings` so cloud vendors (Claude, GPT, Gemini, Grok) and local modes (Player2, llama.cpp) share one selector. Show provider-specific field groups (key, model, endpoint) based on selection without discarding saved values when switching.

## Acceptance Criteria
- [ ] `ProviderMode` (or successor) includes `claude` | `openai` | `gemini` | `grok` | `player2` | `llamacpp`
- [ ] Settings UI uses a `<select>` / accessible dropdown labeled Provider (not radios)
- [ ] Switching provider shows the correct field group; previously saved values for other providers are retained
- [ ] Validation errors are mode-specific (missing key, missing model, bad URL)
- [ ] Save disabled until draft is valid and dirty (existing Save/Cancel behavior preserved)
- [ ] Unit/UI tests cover dropdown change and field visibility
