# Boot failure handling + retry UX

## Description
Design resilient startup failure behavior with retry/recover actions so users are not stranded on startup errors.

## Acceptance Criteria
- [ ] Loading screen handles boot failures with clear error category (DB, runtime, config, unknown)
- [ ] User can trigger retry without full app restart when failure is recoverable
- [ ] Non-recoverable failures provide actionable guidance
- [ ] Failure and retry flows are tested and do not leave orchestrator in invalid state
