# LLM boot stage (launch/health/warmup)

## Description
Add an LLM boot stage that ensures the selected provider runtime is available and ready before AI-backed flows begin.

## Acceptance Criteria
- [ ] Startup stage initializes provider runtime readiness check (llama.cpp-first path)
- [ ] For managed mode, runtime launch + health polling is completed before ready
- [ ] For attach mode, health check validates existing runtime availability
- [ ] Stage emits typed progress/failure events and does not crash app on timeout/unreachable runtime
