# DB migration runner infrastructure

## Description
Build the numbered, forward-only migration runner that `/db` and the rest of the app rely on to create and evolve the SQLite schema.

## Acceptance Criteria
- [ ] A migration runner applies numbered migration files in order against a SQLite file
- [ ] Running the runner twice against the same file is idempotent (already-applied migrations are skipped)
- [ ] A test creates a fresh DB file, runs migrations, and confirms the expected tables exist
- [ ] Opening a DB file that's missing later migrations applies only the pending ones, tested
