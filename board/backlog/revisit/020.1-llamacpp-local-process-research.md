# llama.cpp local-process research spike

## Description
Research-only spike: validate launching llama.cpp's `llama-server` directly as a local process, confirm its readiness/health contract, and confirm its request/response shape maps cleanly onto this app's shared provider interface, before any production adapter code is written. Findings are recorded in [docs/research/llamacpp-local-runtime-2026-06-28.md](../../docs/research/llamacpp-local-runtime-2026-06-28.md) — recommended defaults: `127.0.0.1:8080`, `/v1/chat/completions` (OpenAI-compatible), `/health` for readiness, `--n-gpu-layers all`.

## Acceptance Criteria
- [ ] A research script/prototype (not imported by app runtime code) launches llama.cpp locally and completes at least one generation request
- [ ] Lifecycle behavior is validated and documented: startup, ready check, timeout/failure handling, forced termination, and clean shutdown
- [ ] A minimal prompt/response contract is documented, including how structured (JSON) outputs are requested and validated
- [ ] Model configuration (path/name/options) is documented in a swap-friendly way — no code changes needed to switch models
- [ ] Findings, known risks, and recommended defaults are written up for the follow-on tickets (020.2-020.6)
