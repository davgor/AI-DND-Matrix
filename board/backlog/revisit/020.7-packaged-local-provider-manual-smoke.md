# Packaged local-provider manual smoke

## Description
Run a manual smoke check against a build configured to force the local provider, confirming a real gameplay action succeeds end-to-end and that runtime unavailability fails gracefully and recoverably.

**Gate ticket:** do not run 020.8-020.16 until this passes. Dev-mode (`npm run dev`) is acceptable for first pass; repeat at least once on `release/` packaged build before epic close.

## Prerequisites
- Reference model from [docs/research/llamacpp-local-runtime-2026-06-28.md](../../docs/research/llamacpp-local-runtime-2026-06-28.md) installed and configured
- `AGENT_PROVIDER=llamacpp` (or Settings → llama local mode)
- Managed mode: valid `llamaCppServerPath` + `llamaCppModelPath`; attach mode: `llama-server` already running on configured base URL

## Acceptance Criteria
- [ ] With local provider only (no cloud fallback), a real generation call succeeds end-to-end for a representative gameplay action (e.g. campaign seed or one DM turn)
- [ ] The smoke check is documented as manual, not part of CI
- [ ] Notes capture the exact test setup: runtime availability, model file, config values, build version, and reference model id
- [ ] One expected-failure scenario (runtime missing/unhealthy) is exercised, confirming a typed, user-actionable error and a stable, recoverable app
