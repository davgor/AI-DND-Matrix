# Local process lifecycle manager (llama.cpp)

## Description
Implement a runtime lifecycle manager, owned by the Electron main process, that controls the llama.cpp process's start/stop/health state and isolates its failures from the UI/game loop with typed errors instead of uncaught exceptions.

## Acceptance Criteria
- [ ] The app can launch a configured local llama.cpp process with configured host/port/model settings
- [ ] A readiness/health check (`GET /health`, treating 503 as still-loading rather than fatal) gates first request dispatch
- [ ] Startup failure, timeout, port-conflict, and unexpected-exit paths return typed errors, never uncaught exceptions
- [ ] The lifecycle manager exposes explicit states (e.g. stopped, starting, ready, degraded, stopping) usable by higher layers
- [ ] Calling start() while already starting/ready returns the existing state/handle instead of spawning a duplicate process
- [ ] Shutdown is graceful on app quit/session stop (signal first, forced termination fallback) and leaves no orphaned process; stop() is idempotent
- [ ] Unit tests cover startup success, startup timeout/failure, unexpected process exit, and idempotent stop behavior
