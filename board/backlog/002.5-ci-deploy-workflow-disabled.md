# CI: scaffold disabled deploy-on-merge workflow

## Description
Add a deploy workflow that would build and package a release `.exe` on merge to `main`, but keep it disabled by default until explicitly promoted later.

## Acceptance Criteria
- [ ] `.github/workflows/deploy.yml` exists, triggers on merge to `main`, and runs `npm run package`
- [ ] The workflow is disabled by default (e.g. `workflow_dispatch` only, or a guarded `if:` condition that evaluates false)
- [ ] A comment in the workflow file explains how to flip it on later
