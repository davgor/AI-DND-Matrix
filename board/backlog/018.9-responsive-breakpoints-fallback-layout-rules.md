# Responsive breakpoints and fallback layout rules

## Description
Define how the four-column layout adapts as width decreases so in-campaign mode remains usable on smaller windows.

## Acceptance Criteria
- [ ] Breakpoints define when columns collapse, stack, or become overlay panels
- [ ] Core gameplay flow remains usable when fewer than four columns are visible
- [ ] No column content becomes unreachable due to responsive transitions
- [ ] UI test coverage includes at least desktop four-column and narrowed fallback states
