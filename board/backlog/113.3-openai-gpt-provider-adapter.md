# 113.3 OpenAI GPT provider adapter

## Description
Implement an OpenAI Chat Completions (or current Responses-compatible) provider adapter behind the existing `Provider` interface, configurable by API key + model id from Settings.

## Acceptance Criteria
- [ ] `createOpenAiProvider({ apiKey, model })` implements `Provider.generate`
- [ ] Honors `GenerateContext` (`systemPrompt`, `maxTokens`)
- [ ] Truncation / failure behavior matches project norms (fail loud on max-token truncation where detectable)
- [ ] Config error when API key missing; request errors do not leak the key in messages
- [ ] Unit tests with mocked fetch cover success, auth failure, and truncation/error paths
- [ ] Settings “Test connection” can ping this adapter
