# llama.cpp provider adapter implementation

## Description
Implement the llama.cpp provider adapter against `POST /v1/chat/completions`, satisfying the same shared provider interface the Claude adapter implements and delegating process-readiness concerns to the lifecycle manager from 015.2. Like every provider adapter, it returns generated text only — it never mutates game state itself.

## Acceptance Criteria
- [ ] A llama.cpp adapter implements the shared provider interface used by DM/NPC/party-member agents
- [ ] The adapter maps provider call input to llama.cpp's chat-completions request shape and maps the response back to the shared provider output shape
- [ ] The adapter refuses dispatch when the lifecycle manager (015.2) reports not-ready, surfacing its typed error rather than duplicating lifecycle logic
- [ ] The adapter supports requesting structured (JSON schema-guided) output for downstream validation
- [ ] Malformed/missing response fields return a typed adapter parse error, never an uncaught exception
- [ ] No secrets or sensitive local paths/prompt content are logged in plain text from adapter error paths
- [ ] Unit tests with a mocked local HTTP layer cover a successful generation path and a malformed-output path
