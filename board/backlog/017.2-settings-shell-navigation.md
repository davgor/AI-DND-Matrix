# Settings shell + navigation

## Description
Implement a settings shell/view structure that can host provider setup sections and future settings categories.

## Acceptance Criteria
- [ ] Settings view has stable layout sections (for example: Provider, Runtime, Connectivity, Save/Apply)
- [ ] Navigation into and out of Settings is deterministic (back/cancel/apply behaviors)
- [ ] Unsaved-change behavior is handled intentionally (prompt or auto-save rule)
- [ ] Unit/UI tests cover baseline navigation states
