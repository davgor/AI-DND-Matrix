# Local-provider config wiring + model swap

## Description
Wire local-provider selection and model/runtime settings into config and persisted settings so switching providers or swapping models is a config/settings change, not a code change.

**Current state:** `.env` keys and settings persistence (016) already cover `AGENT_PROVIDER=llamacpp`, base URL, server path, model path, ctx size, GPU layers, and start mode. `resolveProviderRegistryConfig` hydrates these at runtime.

## Acceptance Criteria
- [x] Provider selection supports choosing the local llama.cpp adapter from config/settings only, no code changes required
- [x] Model-specific settings (path, context, GPU layers) are read from config/settings and passed to the lifecycle manager
- [x] Invalid or missing required local-provider config returns typed validation errors at startup, distinguishing config problems from runtime/process problems
- [ ] Unit tests verify provider and model swaps are honored through config-only changes (extend existing `settingsRuntime` / `selectProvider` coverage for path/ctx/gpu fields)

## Deferred (post-v1)
- Default model profile + override profile for per-role model selection (DM/NPC/lore) — config shape can stay flat until a second model is actually needed
