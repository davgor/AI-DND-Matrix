# Region-history compression job

## Description
Implement the job that takes compression candidates (from the db query in 003.4) and summarizes them via the provider into a shorter, compressed region_history entry.

## Acceptance Criteria
- [ ] Given candidate entries, the job calls the provider to summarize them and writes back a new entry with `is_compressed = true`
- [ ] Original detailed entries are excluded from future context assembly once compressed
- [ ] Unit tested against a mock provider
