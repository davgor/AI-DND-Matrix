# Equip/unequip engine logic + slot rules

## Description
Implement the engine logic for equipping and unequipping owned items into the weapon/armor/trinket slots, enforcing one item per slot and that only items with a matching equip slot can be equipped there.

## Acceptance Criteria
- [ ] Equipping an item into a slot that already holds an item automatically unequips the previous one (no two items in the same slot)
- [ ] Attempting to equip an item with no equip slot (e.g. a potion) or a mismatched slot is rejected with a typed result, not a silent no-op or crash
- [ ] Unequipping a slot leaves the item in the character's inventory (not deleted)
- [ ] Equip state is read from and written to the `character_items` table from 024.1
- [ ] Unit tests cover equip, swap-on-equip, unequip, and the invalid-slot rejection path
