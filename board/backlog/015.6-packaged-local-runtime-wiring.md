# Packaged local runtime wiring

## Description
Make packaged builds able to run the local-provider path reliably for downloaded users: resolve the runtime binary/model paths independently of dev-directory assumptions, validate them at startup, and surface clear, actionable errors when they're missing.

## Acceptance Criteria
- [ ] Packaged app can launch the configured local llama.cpp runtime in a production install, not only in dev mode
- [ ] Runtime/model path resolution works in packaged environments without relying on dev-only directory layout
- [ ] Missing runtime or invalid runtime/model path surfaces a clear typed error with a recovery hint
- [ ] A manual packaged-build check confirms a real agent call succeeds through the local-provider path
- [ ] Packaging notes document expected user prerequisites: runtime binary/source, model placement, minimum GPU capability assumptions
