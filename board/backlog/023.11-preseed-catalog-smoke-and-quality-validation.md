# Preseed-catalog smoke and quality validation

## Description
Run end-to-end validation to confirm retrieval-first behavior works and seeded content quality is acceptable.

## Acceptance Criteria
- [ ] Smoke run confirms AI uses preseeded entries when suitable matches exist
- [ ] Smoke run confirms fallback generation when no suitable preseed match exists
- [ ] Validation report includes retrieve-hit rates, fallback rates, and notable quality issues
- [ ] Build/test/lint baseline remains green after catalog integration
