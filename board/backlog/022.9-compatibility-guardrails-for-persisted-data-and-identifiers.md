# Compatibility guardrails for persisted data and identifiers

## Description
Protect runtime/data compatibility when terminology updates intersect with persisted data, IDs, or protocol fields.

## Acceptance Criteria
- [ ] Persisted schema fields/IDs are reviewed to avoid destructive renames that break existing saves
- [ ] Where renames are required, migration or compatibility shims are defined
- [ ] External/public contract strings are versioned or aliased if changed
- [ ] Manual load test verifies older save data still works after scrub
