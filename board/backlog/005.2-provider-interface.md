# Provider interface definition

## Description
Define the provider-agnostic interface that DM/NPC/party-member agents call, so Player2, Claude, or others can implement it interchangeably.

## Acceptance Criteria
- [ ] `/agents/providers/types.ts` defines a `generate(prompt, context) -> text` (or equivalent) interface with no Player2-specific details leaking into it
- [ ] A trivial mock implementation satisfies the interface and is usable in tests
