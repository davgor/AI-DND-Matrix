# Context assembly recency-window enforcement

## Description
Enforce the recency-window + relevance-tag rule across all agent context assembly: bounded number of memory/event entries, never a full-history dump.

## Acceptance Criteria
- [ ] Context assembly caps the number of npc_memories/region_history/events entries included, regardless of how much history exists
- [ ] Unit tested with a large seeded history (e.g. hundreds of entries) confirming the assembled context stays within the defined bound
