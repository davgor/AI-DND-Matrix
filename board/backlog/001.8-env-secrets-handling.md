# .env / secrets handling

## Description
Set up gitignored local secrets/config via `.env`, loaded by the main process at startup, with a checked-in `.env.example` documenting expected keys.

## Acceptance Criteria
- [ ] `.env` is listed in `.gitignore`
- [ ] `.env.example` exists, documenting expected keys (provider selection, Claude API key placeholder, etc.)
- [ ] The main process loads `.env` at startup and the loaded values are not exposed to the renderer process
