# Party-member routing alignment

## Description
Align AI party-member actions with the new routing model: companions react when the plan or scene warrants it, not automatically after every DM narration. Reuse `decidePartyMemberAction` with context that reflects which beats already fired this turn.

## Acceptance Criteria
- [ ] Party-member agent does not run on converse-only NPC turns unless the plan includes a party beat
- [ ] Party lines still append `party_member_action` events and render in the exposition feed
- [ ] Unit tests cover party silent on dialogue turn and party action on combat/narration turn
