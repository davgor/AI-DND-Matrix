# 118.4 — Wire play view `lastCheck` to D20 overlay

## Description

Connect the overlay to live turn outcomes: when `usePlayViewController` / turn submit sets a new `lastCheck` with a d20 `roll`, start the overlay using 118.1 policy. Do not re-trigger on first mount with a hydrated/stale check.

Keep the existing Scene-feed `formatRoll` line when Show rolls is on; overlay is additive spectacle.

## Acceptance criteria

- [ ] New live `lastCheck` with `roll` starts the D20 overlay in the in-campaign play view
- [ ] Remount / first paint with an existing `lastCheck` does **not** animate
- [ ] Show-rolls policy from 118.1 applied (face persistence / clear behavior)
- [ ] Non-check turns (no `check`) do not flash the die
- [ ] Integration or component test with fake timers covers fire-on-change and no-fire-on-mount
