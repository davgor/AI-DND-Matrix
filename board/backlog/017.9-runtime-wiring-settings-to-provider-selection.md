# Runtime wiring from settings to provider selection

## Description
Wire persisted settings into runtime provider selection so agent calls use the configured provider mode and options.

## Acceptance Criteria
- [ ] Provider registry/provider builder consumes hydrated settings rather than hardcoded/default-only config
- [ ] Switching provider mode from settings changes runtime provider behavior without code changes
- [ ] Invalid configured provider state surfaces typed runtime error rather than uncaught exception
- [ ] Unit tests verify provider selection for API-key, llama local, and Player2 modes
