# Compatibility, migration, and data integrity checks

## Description
Protect existing campaigns/saves while introducing preseed catalog data and retrieval-first logic.

## Acceptance Criteria
- [ ] Existing save/campaign data remains valid after schema and retrieval updates
- [ ] Migrations are backward-safe and validated on upgrade paths
- [ ] Integrity checks catch orphaned tags/mappings and duplicate canonical keys
- [ ] Recovery guidance exists for integrity failures
