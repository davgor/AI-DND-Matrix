# 113.7 Registry wiring, secrets, validation, and tests

## Description
Wire new providers into `createProviderRegistry` / `selectProvider`, persist and redact API keys for GPT/Gemini/Grok like Claude, and harden validation + migration from the pre-113 settings shape.

## Acceptance Criteria
- [ ] Runtime selects the correct adapter from saved settings mode + model

- [ ] `.env` / bootstrap story documented for new keys (or intentionally Settings-only — stated in ticket notes)
- [ ] Old settings JSON without new fields loads with safe defaults (no crash)
- [ ] Integration-style tests cover registry selection for each new mode
- [ ] oxlint-clean; no `any` escapes in new code
