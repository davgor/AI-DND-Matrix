# Duplicate-submit and idempotency protection

## Description
Prevent accidental duplicate campaigns from repeated submit clicks or retried network responses.

## Acceptance Criteria
- [ ] Client-side submit lock prevents repeated in-flight submissions
- [ ] Main process flow guards against duplicate create requests for same in-flight session
- [ ] Malformed/failing attempts do not leave partial campaign rows
- [ ] Tests verify one successful campaign is persisted per successful submit action
