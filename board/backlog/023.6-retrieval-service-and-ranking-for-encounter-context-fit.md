# Retrieval service and ranking for encounter/context fit

## Description
Implement catalog retrieval and ranking so runtime systems can fetch best-fit preseeded entries.

## Acceptance Criteria
- [ ] Retrieval API supports filters by bucket, encounter context, level/difficulty, and tags
- [ ] Ranking strategy prioritizes context fit and diversity constraints
- [ ] Retrieval returns deterministic results for identical inputs unless randomness is explicitly configured
- [ ] Unit tests cover ranking, filtering, and no-match behavior
