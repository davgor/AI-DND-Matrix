# Log book DB schema + repository

## Description
Add a `log_entries` table scoped to a character (campaign + character id), with a category enum (`event`/`place`/`person`/`beast`/`thing`), title, content, an optional loose reference to the related entity (NPC/region/item, not FK-enforced since it varies by category), and the in-game date it was learned.

## Acceptance Criteria
- [ ] `log_entries` table stores campaign id, character id, category (constrained to the five valid values), title, content, optional related-entity id, in-game date learned, and created-at timestamp
- [ ] Repository functions exist for: create an entry, list entries for a character, list entries for a character filtered by category
- [ ] Querying one character's log entries never returns another character's entries (isolation), mirroring the existing `npc_memories` isolation guarantee
- [ ] Unit tests cover create/list/filter-by-category and the isolation guarantee
