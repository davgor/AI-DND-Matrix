# Targeted NPC response path

## Description
When the routing plan selects NPC response, invoke `generateNpcReaction` for the targeted NPC(s) — including the case where the player is only talking to someone and DM narration is minimal or skipped. Remove the requirement that `reactingNpcIds` only come from the narration JSON. NPC context stays isolated per epic 006; speaking vs non-speaking branches stay per epic 028.

## Acceptance Criteria
- [ ] NPC agent runs only for ids in the routing plan (not every present NPC by default)
- [ ] Converse-first turn can return NPC dialogue without a preceding scene paragraph when the plan says so
- [ ] `npc_reaction` events record `reactionKind`, text, and optional attack flag as today
- [ ] Unit tests cover targeted single-NPC dialogue and non-speaking creature action response
