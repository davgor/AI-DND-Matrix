# llama.cpp local-process research spike

## Description
Research-only spike: validate launching llama.cpp's `llama-server` directly as a local process, confirm its readiness/health contract, and confirm its request/response shape maps cleanly onto this app's shared provider interface, before any production adapter code is written. Findings are recorded in [docs/research/llamacpp-local-runtime-2026-06-28.md](../../docs/research/llamacpp-local-runtime-2026-06-28.md) — recommended defaults: `127.0.0.1:8080`, `/v1/chat/completions` (OpenAI-compatible), `/health` for readiness, `--n-gpu-layers all`.

## Acceptance Criteria
- [x] Lifecycle behavior is validated and documented: startup, ready check, timeout/failure handling, forced termination, and clean shutdown
- [x] A minimal prompt/response contract is documented, including how agent layers parse JSON from free-form model text (prompt + `tryParseJson`, not provider JSON-schema mode)
- [x] Model configuration (path/name/options) is documented in a swap-friendly way — no code changes needed to switch models
- [x] Findings, known risks, and recommended defaults are written up for the follow-on tickets (020.2-020.6)
- [ ] A research script/prototype (not imported by app runtime code) launches llama.cpp locally and completes at least one generation request — **optional for close-out**; manual curl or `llama-server` + Settings connectivity check is sufficient if documented in the research note

## Notes
Research spike is effectively complete for planning purposes. Remaining optional item is a standalone script under `scripts/` or `docs/research/` for repeatability — not a blocker for 020.2+.
