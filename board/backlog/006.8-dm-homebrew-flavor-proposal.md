# DM agent: homebrew flavor proposal call

## Description
At level-up, when the engine's emergent-direction detection (004.22) returns a candidate, call the DM agent to propose flavor (name/description/damage type) constrained to that level's mechanical template (004.23).

## Acceptance Criteria
- [ ] When a candidate exists, the DM agent is called and its response is constrained to flavor-only fields (no numeric override accepted)
- [ ] When no candidate exists, no homebrew call is made
- [ ] Both cases unit tested with a mock provider
