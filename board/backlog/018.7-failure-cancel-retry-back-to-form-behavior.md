# Failure, cancel, retry, and back-to-form behavior

## Description
Handle failures and user cancellation gracefully, allowing retries or form edits without app restart.

## Acceptance Criteria
- [ ] Failed generation/persistence shows clear actionable error in modal flow
- [ ] User can return to form with previous inputs preserved
- [ ] User can retry from loading/error state without creating duplicate partial campaign data
- [ ] Canceling flow does not create partial campaign records
