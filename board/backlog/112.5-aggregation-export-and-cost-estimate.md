# 112.5 Aggregation, export, and cost estimate

## Description
Add rollups and a configurable price table so token totals become estimated USD per purpose/bucket — inputs for subscription tier design.

## Acceptance Criteria
- [ ] Aggregation returns totals by purpose and by bucket (`setup` vs `play` vs `meta`) for a time range and optional campaign
- [ ] Configurable price table maps provider+model → $/1M input and $/1M output tokens (defaults for current Claude model; local/Player2 default $0)
- [ ] Estimated cost = f(tokens, price table); missing usage yields “unknown” rather than $0 silently when tokens are null
- [ ] Export path exists (JSON and/or CSV via IPC/settings debug) suitable for spreadsheet analysis
- [ ] Minimal UI or settings panel section shows recent totals (can be developer-facing v1)
- [ ] Unit tests cover aggregation math and price-table application
