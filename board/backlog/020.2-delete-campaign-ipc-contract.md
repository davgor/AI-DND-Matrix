# Delete-campaign IPC contract

## Description
Add a typed `campaigns:delete` IPC channel (preload-exposed, narrow, no generic exec/SQL surface) that the confirmation UI from 020.1 calls on confirm, returning a typed success/failure result.

## Acceptance Criteria
- [ ] A typed `campaigns:delete(campaignId)` IPC channel exists, exposed through preload the same way other campaign channels are
- [ ] Main-process handler validates the campaign exists before attempting deletion and returns a typed not-found error if it doesn't
- [ ] Handler returns a typed success result on completion and a typed failure result (not an uncaught exception) if deletion fails partway through
- [ ] Unit tests cover the success path and the not-found/failure paths
