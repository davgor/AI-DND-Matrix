# Local-provider retry/backoff + failure logging

## Description
Add retry-with-backoff for transient local-provider failures and a typed terminal "unreachable" error once retries are exhausted, logging failures to the existing local log pipeline with parity to how the Claude adapter already handles retries/logging.

## Acceptance Criteria
- [ ] Failing local-provider calls are retried a small fixed number of times with backoff before giving up
- [ ] Non-retryable errors (e.g. invalid config) fail fast without delay
- [ ] After retries are exhausted, a typed local-provider-unreachable error is surfaced, not an uncaught exception
- [ ] Failure events are written to the local log file with sanitized diagnostics (provider name, host/port, lifecycle state, retry count, error class) and no secrets
- [ ] Unit tests cover transient failure then success, repeated failure then typed terminal error, and immediate fail-fast on non-retryable config errors
