# Active character selection through play shell

## Description
Plumb selected `characterId` through `PlayView`, turn IPC, narration log, character sheet, journal, log book, level-up modal, and `findPlayerCharacter` call sites. Session recap, saves, and campaign-level world mutations remain shared; narration log queries filter by active `characterId`.

Replace `findPlayerCharacter(characters)` with explicit active-character id from app state everywhere play behavior depends on "the" player.

Depends on **038.6**.

## Acceptance Criteria
- [ ] `PlayView` receives and uses explicit `characterId` — not first player row
- [ ] Turn submission, narration log, sheet, journal, and log book all scope to active `characterId`
- [ ] Switching characters from hub loads the correct per-character state (region, HP, inventory, log book)
- [ ] Unit/integration tests prove two player characters in one campaign do not leak log entries or narration between them
