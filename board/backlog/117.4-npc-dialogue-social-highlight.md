# 117.4 — Highlight new NPC dialogue in Social stream

## Description

When an NPC speaks (`speaker === 'npc'`, `reactionKind === 'dialogue'`) and that line is projected into the Social stream (`filterSocialEntries` / `SocialMessage`), apply the shared incoming highlight to that message bubble (or message row) for a few seconds so players notice speech that arrived while they were watching Scene or the sheet rail.

## Acceptance criteria

- [ ] Newly appended NPC dialogue Social messages glow for the shared duration
- [ ] Player messages, party-member lines, and NPC/creature **action** reactions do **not** glow (v1)
- [ ] Mount / history window load does **not** glow existing NPC lines; only live appends after mount
- [ ] Auto-scroll / window paging behavior from **085** still works; highlight does not break layout
- [ ] Component test: append NPC dialogue → highlighted; append player / action → not highlighted
