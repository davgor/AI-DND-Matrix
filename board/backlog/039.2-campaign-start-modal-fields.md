# Campaign start modal: region + NPC count fields

## Description
Add two controls to the new-campaign setup modal (`CampaignStartFormFields`):

1. **Regions to generate** — integer 0–5, default 2
2. **NPCs per region** — integer 0–10, default 3

Use the same UX patterns as death mode (labeled inputs, muted hint text explaining what the numbers mean). Disable while submitting. Wire into `useCampaignStartFlow` form state.

Depends on **039.1**.

## Acceptance Criteria
- [ ] Modal shows both fields with correct min/max/default values on first open
- [ ] User can change values within bounds; out-of-range input is clamped or blocked with inline validation
- [ ] Hint text explains that 0 is allowed but review will require at least 1 region and 1 NPC before continuing
- [ ] Form reset on modal close/reopen restores defaults
- [ ] UI test or component test verifies defaults and bound enforcement
