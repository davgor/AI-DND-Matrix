# Startup boot contract + readiness states

## Description
Define the startup contract and stage model used by main/preload/renderer so loading behavior is deterministic and testable.

## Acceptance Criteria
- [ ] A typed startup state model exists (for example: idle, booting, waitingDb, waitingLlm, ready, failed)
- [ ] Required readiness gates are defined and documented for DB and provider runtime
- [ ] Startup emits typed progress/status events over IPC with stable payload shape
- [ ] Unit tests validate state transitions and illegal transition rejection
