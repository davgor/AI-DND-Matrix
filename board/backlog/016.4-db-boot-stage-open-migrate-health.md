# DB boot stage (open/migrate/health)

## Description
Add a database boot stage that validates DB readiness before gameplay routes are enabled.

## Acceptance Criteria
- [ ] Startup stage opens DB connection and runs required migrations/checks
- [ ] Stage reports progress and completion/failure back to orchestrator
- [ ] Failure path returns typed boot error with user-actionable message
- [ ] Stage is covered by tests for success and migration/connection failure
