# Local-provider config wiring + model swap

## Description
Wire local-provider selection and model/runtime settings (e.g. `AGENT_PROVIDER=llamacpp`, `LLAMA_CPP_BASE_URL`, `LLAMA_CPP_MODEL_PATH`, `LLAMA_CPP_SERVER_PATH`, `LLAMA_CPP_CTX_SIZE`, `LLAMA_CPP_GPU_LAYERS`) into config, so switching providers or swapping models is a config-only change. Config should support a default model profile plus at least one override profile, so role-based model selection (DM/NPC/lore) can be added later without a config-shape rewrite.

## Acceptance Criteria
- [ ] Provider selection supports choosing the local llama.cpp adapter from config/.env only, no code changes required
- [ ] Model-specific settings (path/name/context/temperature) are read from config and passed through the local adapter
- [ ] Config supports a default model profile and an override profile
- [ ] Invalid or missing required local-provider config returns typed validation errors at startup, distinguishing config problems from runtime/process problems
- [ ] Unit tests verify provider and model swaps are honored through config-only changes
