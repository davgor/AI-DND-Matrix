# Character journal DB schema + repository

## Description
Add a simple `character_journal_entries` table scoped to a character (campaign + character id): free-text content, in-game date, created-at — no categories, no structured fields, this is a flat diary feed.

## Acceptance Criteria
- [ ] `character_journal_entries` table stores campaign id, character id, free-text content, in-game date, and created-at timestamp
- [ ] Repository functions exist for: create an entry, list a character's entries in reverse-chronological order
- [ ] Querying one character's journal never returns another character's entries
- [ ] Unit tests cover create/list-ordering and the isolation guarantee
