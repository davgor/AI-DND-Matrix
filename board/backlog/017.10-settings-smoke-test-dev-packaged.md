# Settings smoke test (dev + packaged)

## Description
Validate end-to-end settings behavior for provider setup in development and packaged builds.

## Acceptance Criteria
- [ ] Dev-mode smoke verifies: open settings from main screen, update provider settings, apply, and observe runtime provider change
- [ ] Packaged-mode smoke verifies same flows for at least one API-key mode and one local/endpoint mode
- [ ] Expected-failure smoke verifies actionable errors for invalid key/runtime/endpoint config
- [ ] Smoke runbook documents steps, environment, and observed outcomes
