# Travel resolution: in-game date advance

## Description
Implement travel-time resolution: advances the campaign's in-game date by a DM-estimated number of days, clamped to a sane range by the engine.

## Acceptance Criteria
- [ ] `resolveTravel(estimatedDays)` clamps the input to a defined sane range (e.g. 0-30 days) before returning the actual advance
- [ ] Unit tested for a normal value (passes through), a too-large value (clamped down), and a negative value (rejected or clamped to 0)
