# 020.27 — Stop managed llama-server before runtime re-acquire

## Description

Acquiring a new llama.cpp runtime (e.g. switching CPU → Vulkan GPU) while a managed `llama-server` is still running fails with `EBUSY: resource busy or locked` when overwriting DLLs such as `ggml-base.dll`. Stop the managed lifecycle before replacing runtime files so the install can complete, then leave the process stopped for the next Apply / Check runtime / first-run start.

Extends epic 020 Local llama.cpp provider (follows 020.23 GPU backend + 149 full-package acquire).
