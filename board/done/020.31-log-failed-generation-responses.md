# 020.31 — Log failed campaign-generation LLM responses

World (and other stage) schema failures currently only surface the message “DM agent did not return a valid world schema after retries,” with no visibility into what the local model actually returned. Debugging 7B shape/parse issues needs the raw attempt payloads in the main log.

## Acceptance criteria

- [x] `CampaignGenerationSchemaError` carries per-attempt raw response + failure reason (`unparseable` / `normalize_failed` / `invalid`)
- [x] `generateWithRetries` records each failed attempt (unit-tested via `generateCampaignWorld`)
- [x] `createCampaignFromRequest` logs those attempt details via the main logger (unit-tested with injectable/spy logger or extracted formatter)
- [x] Logged raw text is truncated to a safe max length (no multi-MB dumps)
- [x] `npm test` / `lint` / `build` / `deadcode` + act pr-checks + deadcode workflows pass
