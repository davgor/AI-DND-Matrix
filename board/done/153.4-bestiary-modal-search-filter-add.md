# 153.4 — Bestiary: summary + modal with defaults, search/filter, and prompt Add

## Description

Match pantheon/factions: campaign page shows a short bestiary summary and **View Bestiary** opens a content-width modal. The modal must include **pre-injected catalog (default) enemies** plus **campaign-specific** species, with text search and a filter for default vs campaign. An **Add** control generates a customized campaign species from a user prompt (reuse `generateOrGetBestiarySpecies`) and refreshes campaign detail.

## Acceptance criteria

- [x] Page shows bestiary summary chrome + **View Bestiary** (not the full inline dump)
- [x] Modal lists catalog seed creatures as `origin: 'default'` and campaign species as `origin: 'campaign'`
- [x] Modal supports text search and filter: All / Default enemies / Campaign-specific
- [x] **Add** opens a seed-prompt flow; successful generate persists a campaign species and updates detail
- [x] Unit tests cover roster merge/filter, summary chrome, modal search/filter, and generate IPC happy path
- [x] `npm test` / `npm run lint` / `npm run build` / `npm run deadcode` pass
- [x] `act` pr-checks.yml + deadcode.yml succeed
