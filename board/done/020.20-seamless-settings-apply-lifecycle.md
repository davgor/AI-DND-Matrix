# Seamless Apply from Settings (boot + lifecycle)

## Description
Close the loop so choosing Local + a ready downloaded model (+ runtime) and clicking **Save/Apply** results in a ready local provider for gameplay — without editing `.env`, without restarting into a broken boot that ignores Settings, and without requiring a separate “check runtime” ritual that always fails.

Wire persisted Settings (mode, start mode, resolved server/model paths, ctx/gpu) into the startup LLM boot stage and into an explicit post-save apply/restart-lifecycle path.

## Acceptance Criteria
- [x] Startup LLM boot stage reads llama lifecycle config from **persisted settings** (with `.env` as override/fallback only), including managed paths written by 020.18–020.19
- [x] After Save/Apply in Settings with Local mode + ready model + runtime, lifecycle reaches `ready` (or surfaces a typed failure) without requiring a full app reinstall or manual `.env` edits
- [x] Switching away from Local stops or detaches cleanly; switching back to a ready local setup can start again without orphan processes in the common path
- [x] Managed “Check runtime” / readiness checks use real filesystem existence checks in production (fix the pathExists-always-false gap)
- [x] Unit tests cover settings→lifecycle config mapping and at least one apply/start success + one missing-asset failure path

## Notes
- Coordinates with epic 015 loading UI for first launch; mid-session Apply may show a compact Settings progress state rather than the full boot splash.
- Agent provider selection must continue to honor the same persisted mode/base URL after Apply.
