# 020.30 — Fail fast on world schema failure (no ×5 seed restart)

After 020.29, local campaign create still sits on **World** for a long time and “restarts.” Cause: `CampaignGenerationSchemaError` from world/pantheon/etc. is swallowed by `tryCampaignSeedAttempt`, so the outer seed loop redoes canon→pantheon→world up to `MAX_CAMPAIGN_SEED_ATTEMPTS` (5). Each world call on Qwen 7B with a large `max_tokens` is wall-clock heavy; nested retries burn minutes, then surface the generic “invalid campaign generation schema” error.
