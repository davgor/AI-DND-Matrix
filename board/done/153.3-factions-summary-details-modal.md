# 153.3 — Factions: summary on page, details in modal

## Description

The campaign factions block currently dumps roster + relations inline under the high-level summary. Match the pantheon pattern: keep **pressure + summary** (and edit) on the page, and open a full-width details modal via **View Factions** that surfaces each faction’s rich fields (`summary`, `motivation`, `publicFace`, `methods`, kind, deity) plus relation stances (and relation summaries when present).

## Acceptance criteria

- [x] Campaign factions section shows pressure + summary only (no inline roster/relations list)
- [x] **View Factions** opens a content-width modal (same layout pattern as pantheon)
- [x] Modal lists each faction with available detail fields, not just name/kind
- [x] Modal includes relations readout (stance + optional relation summary)
- [x] Unit tests cover summary-only section chrome and rich modal content
- [x] `npm test` / `npm run lint` / `npm run build` / `npm run deadcode` pass
- [x] `act` pr-checks.yml + deadcode.yml succeed
