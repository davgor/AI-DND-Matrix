# Packaged local-provider manual smoke (download happy path)

**Status: Dropped.** Epic 020 closed without a formal manual smoke pass; the local provider shipped on the strength of 020.1–020.33. This checklist was never executed.

## Description
Run a manual smoke check that exercises the **rescoped** local-provider UX: Settings → pick reference catalog model → download (and acquire runtime if needed) → Apply → one real gameplay generation succeeds; also confirm a graceful expected-failure path.

**Gate ticket:** do not run 020.8–020.16 until this passes. Dev-mode (`npm run dev`) is acceptable for first pass; repeat at least once on a `release/` packaged build before epic close.

## Prerequisites
- 020.17–020.20 implemented enough for the happy path (catalog, download, runtime acquire, Settings → lifecycle)
- Reference catalog entry from [docs/research/llamacpp-local-runtime-2026-06-28.md](../../docs/research/llamacpp-local-runtime-2026-06-28.md) (Qwen2.5-7B-Instruct Q4_K_M or the pinned successor)
- No requirement to pre-install a `.gguf` by hand for the happy-path run
- Advanced BYO paths may be used only for the expected-failure / recovery notes, not as the primary proof

## Acceptance Criteria
- [ ] Happy path: Settings → Local → select reference model → Download completes → Apply → local provider ready, with no manual path pasting and no `.env` edits
- [ ] With local provider only (no cloud fallback), a real generation call succeeds end-to-end for a representative gameplay action (e.g. campaign seed or one DM turn)
- [ ] The smoke check is documented as manual, not part of CI
- [ ] Notes capture: build version, catalog model id, whether runtime was discovered vs acquired, `userData` paths used, and any VRAM/RAM notes
- [ ] One expected-failure scenario (runtime missing, download incomplete, or unhealthy server) is exercised, confirming a typed, user-actionable error and a stable, recoverable app via Settings
