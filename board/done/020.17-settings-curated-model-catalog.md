# Settings curated model catalog

## Description
Replace the primary local-model UX (raw path text fields) with a **curated catalog** in Settings: the user picks a recommended model to try, sees size/VRAM guidance, and chooses Download / Use. Manual path entry remains available as an advanced/collapsed escape hatch.

Depends on config keys from 020.4; download/apply behavior is 020.18–020.20. This ticket owns catalog data + Settings UI selection state.

## Acceptance Criteria
- [x] Settings Local section shows a curated list of at least one recommended GGUF profile (pinned reference: Qwen2.5-7B-Instruct Q4_K_M) with human-readable name, approximate download size, and VRAM/RAM hint
- [x] User can select a catalog entry as the active target model; selection persists in settings
- [x] UI distinguishes states: not downloaded / downloading / ready / failed (wired to 020.18 progress events when available; stubbed states acceptable until 020.18 lands if tests cover the state machine)
- [x] Advanced manual server/model path controls remain reachable but are not the default primary path
- [x] Unit/component tests cover catalog rendering, selection persistence, and advanced-section still validating manual paths when used

## Notes
- Catalog may be a static in-repo manifest for v1 (URL + checksum + quant metadata); live remote catalog index is post-v1.
- Do not block Save entirely on “Check runtime” once 020.20 provides Apply/start — align gate with download+runtime readiness instead of path-only checks.
