# 020.28 — Runtime backend GPU/CPU radio buttons

## Description

Replace the first-run onboarding and Settings Local **GPU (Vulkan) / CPU** checkboxes with mutually exclusive **radio** controls styled to match the app’s existing radio patterns (same exclusivity semantics as campaign-start / ability-method radios; theme accent instead of browser-default checkbox chrome). Selection still drives `llamaCppRuntimeBackend` / first-run backend choice.

Follow-up to 020.24 and 020.25.
