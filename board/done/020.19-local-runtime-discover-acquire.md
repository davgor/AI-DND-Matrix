# Local runtime discover / acquire (llama-server)

## Description
Make the happy path work when the user has no `llama-server` on PATH yet: discover an existing install when present, otherwise **acquire** a platform-appropriate runtime binary into `userData` (or guide a one-click install flow), then persist the resolved server path for managed mode.

This is not “bundle llama-server inside the game installer”; it is on-demand acquire parallel to model download (020.18).

## Acceptance Criteria
- [x] On Local provider setup, the app detects a usable `llama-server` via PATH and/or last-known `userData` runtime path and surfaces that state in Settings
- [x] If no runtime is found, the user can acquire one from Settings (download official release into `userData`, or an equivalent automated install path documented for Windows first)
- [x] Acquired/discovered path is persisted and used by managed lifecycle without manual path pasting
- [x] Failure to acquire/discover returns a typed error with recovery hints (retry acquire, install instructions fallback)
- [x] Unit tests cover: PATH hit, `userData` hit, miss → acquire success, acquire failure (mocked)
- [x] winget / GitHub manual install remains documented as advanced fallback only

## Notes
- Windows is the v1 target platform for acquire; other OS can keep discover + manual fallback if acquire is not ready.
- Attach mode (user-managed external server) remains supported and does not require acquire.
