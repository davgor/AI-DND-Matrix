# In-app model download manager

## Description
Implement main-process download of curated GGUF weights into the `userData` layout from 020.6, with progress, cancel, integrity check, and persistence of the resolved model path into settings so 020.20 can start the lifecycle without manual file placement.

Renderer never performs the download; IPC exposes start/cancel/progress/status only.

## Acceptance Criteria
- [x] User can start a download for the selected catalog model from Settings; bytes are written under the documented `userData` models directory
- [x] Progress (bytes received / total or percent) and terminal states (complete, failed, cancelled) are pushed to the renderer over IPC
- [x] Cancel stops the transfer and does not leave a “ready” model path pointing at a partial file (incomplete files are discarded or marked non-ready)
- [x] On success, settings gain a resolved absolute model path (and catalog id) used by managed lifecycle
- [x] Failed downloads surface a typed, actionable error (network, disk, checksum mismatch) without crashing the app
- [x] Unit tests cover success, cancel, checksum/integrity failure, and “do not mark ready on partial file” behavior (mocked HTTP/fs)

## Notes
- Source URLs come from the 020.17 catalog manifest.
- Resume support is nice-to-have; v1 may restart cleanly after cancel/fail.
- App auto-update (`autoUpdate.ts`) must not be reused as the model downloader — keep a dedicated llamacpp/model download module.
