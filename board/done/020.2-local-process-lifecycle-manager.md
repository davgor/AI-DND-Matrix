# Local process lifecycle manager (llama.cpp)

## Description
Implement a runtime lifecycle manager, owned by the Electron main process, that controls the llama.cpp process's start/stop/health state and isolates its failures from the UI/game loop with typed errors instead of uncaught exceptions.

Implemented in `src/main/llamacpp/lifecycle.ts` and wired from `src/main/startupIpc.ts` / `createLlmBootStage`.

## Acceptance Criteria
- [x] The app can launch a configured local llama.cpp process with configured host/port/model settings
- [x] A readiness/health check (`GET /health`, treating 503 as still-loading rather than fatal) gates first request dispatch
- [x] Startup failure, timeout, and unexpected-exit paths return typed errors, never uncaught exceptions
- [x] The lifecycle manager exposes explicit states (stopped, starting, ready, degraded, stopping) usable by higher layers
- [x] Calling start() while already starting/ready returns the existing state/handle instead of spawning a duplicate process
- [x] Shutdown is idempotent on app quit/session stop and leaves no orphaned process in the common path
- [x] Unit tests cover startup success, startup timeout/failure, concurrent start deduplication, 503-then-200 polling, and idempotent stop

## Remaining (non-blocking for 020.3)

- [x] Forced termination fallback if SIGTERM does not exit within a short grace window
