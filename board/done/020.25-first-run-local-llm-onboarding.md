# 020.25 — First-run local LLM onboarding (dev always first-time)

## Description

Change the opening Settings Intro into a local-LLM first-run flow: ask whether to download a local model, then GPU vs CPU, then immediately acquire runtime + download the reference catalog model and switch to managed llama.cpp. On unpackaged **dev builds**, always treat the user as first-time (force-show intro; do not persist dismiss).

Depends on 020.18–020.24 (download, acquire, Vulkan/CPU backend).

## Acceptance criteria

- [x] Unpackaged/dev builds always show the intro (`devForceShow`), even after dismiss; packaged builds still honor persisted dismiss
- [x] Opening prompt asks whether to download a local LLM; Yes → GPU/CPU choice → automatic runtime acquire + catalog model download + mode switch to `llamacpp` managed
- [x] Progress (and failure with recovery) is shown during setup; No → existing provider-setup fallback (Got it / Open Settings)
- [x] Unit tests cover: always-show-in-dev, intro step wiring, and setup orchestration (mocked IPC)
- [x] `npm test`, `npm run lint`, `npm run build`, `npm run deadcode`, and act CI pass
