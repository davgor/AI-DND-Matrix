# 153.1 — Remove generative tokens toggle from campaign review

## Description

The campaign review page currently shows **"Use generative tokens?"**, which lets players flip the flag after world generation. Product rule: that preference is chosen only at **campaign start**. Remove the review checkbox (and its saver wiring); keep the start-form control and persistence at create time.

## Acceptance criteria

- [x] Campaign review UI does not render the "Use generative tokens?" checkbox
- [x] Campaign start form still exposes "Use generative tokens?" (default OFF)
- [x] Review no longer calls `editGenerativeTokens` from the page savers
- [x] Unit test covers that review does not mount the toggle
- [x] `npm test` / `npm run lint` / `npm run build` / `npm run deadcode` pass
- [x] `act` pr-checks.yml + deadcode.yml succeed
